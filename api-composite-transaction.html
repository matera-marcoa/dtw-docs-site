Digital Twin Composite Transaction API

These endpoints deal with multiple operations like credit and debit to different accounts within the same transaction. They also deal with steps related with monitoring these transactions.

Base URL

http://localhost:8080/v1

POST /transaction-bundles

Add transactions into different accounts in a bundle operation

A bundle represents the consolidation of both credits and debits occurring across various accounts, culminating in a single transaction. This request requires a logged user with the *transact* role on both dtw-composite-transaction and dtw-transaction modules.

DELETE /transaction-bundles/{bundle}

Delete transactions from different accounts in a bundle operation

This request requires a logged user with the *transact* role on both dtw-composite-transaction and dtw-transaction modules.

Parameters

Name Type In Required Description
bundle N/A path Yes The unique identifier of the bundle operation.

POST /accounts/{branch}/{account}/monitorings/{monitoringId}/release

Release a monitored sum into an account

It's possible to debit a sum in the same request by filling in the field "blueprints." This request requires a logged user with the *release-monitorings*, *view-monitorings*, and *modify-monitorings* on dtw-transaction and the role *release-monitorings* on dtw-composite-transaction.

Parameters

Name Type In Required Description
if-match N/A header Yes The last registered version for the monitoring service.

POST /account-groups/{kind}/{membershipId}/monitorings/{monitoringId}/release

Release a monitored sum into a group of accounts

It's possible to debit a sum in the same request by filling in the field "blueprints." This request requires a logged user with the *release-monitorings*, *view-monitorings*, and *modify-monitorings* on dtw-transaction and the role *release-monitorings* on dtw-composite-transaction.

Parameters

Name Type In Required Description
if-match N/A header Yes The last registered version for the monitoring service.
