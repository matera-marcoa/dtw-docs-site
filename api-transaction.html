Digital Twin Transaction Services API

These endpoints credit and debit accounts on Digital Twin, approve/decline transaction requests, update limits and balances as well as set up monitoring for credits to accounts.

Base URL

http://localhost:8080

POST /v1/accounts/{branch}/{account}/batches

Add transactions to an account

This request approves or declines transactions as a group. It requires a logged user with the *transact* role assigned.

Parameters

Name Type In Required Description
N/A No
N/A No

DELETE /v1/accounts/{branch}/{account}/entries

Delete transactions from an account

This request requires a logged user with the *transact* role assigned.

GET /v1/accounts/{branch}/{account}/monitorings

List monitoring services setup for the account

This request requires a logged user with the *view-monitorings* role assigned.

Parameters

Name Type In Required Description
N/A No
N/A No
correlationIds array query No A list of correlation IDs representing the monitoring services.

POST /v1/accounts/{branch}/{account}/monitorings

Set up monitoring service for an account

This request requires a logged user with the *modify-monitorings* role assigned.

PATCH /v1/accounts/{branch}/{account}/monitorings

Update monitoring service

This request requires a logged user with the *modify-monitorings* role assigned.

Parameters

Name Type In Required Description
if-match N/A header Yes The current monitoring registry version. This parameter value has to be equal to the last monitoring version.
correlationId array query No Correlation ID

GET /v1/accounts/{branch}/{account}/monitorings/{id}/entries

List monitored transactions for an account

This request requires a logged user with the *view-monitorings* role assigned.

GET /v1/accounts/{branch}/{account}/monitorings/{id}

Find monitoring service for an account by ID

This request requires a logged user with the *view-monitorings* role assigned.

PATCH /v1/accounts/{branch}/{account}/monitorings/{id}

Update a monitoring associated with an account.

This request requires a logged user with the *modify-monitorings* role assigned.

Parameters

Name Type In Required Description
if-match N/A header Yes The current monitoring registry version. This parameter value has to be equal to the last monitoring version.

POST /v1/accounts/{branch}/{account}/monitorings/cancel

Cancel a monitoring service into an account.

Cancel a monitoring service related to an account. > **Important:** This operation requires a logged user with the *modify-monitorings* role

POST /v1/accounts/{branch}/{account}/monitorings/{id}/release

Release a monitored sum into an account

It's possible to debit a sum in the same request by filling in the field "blueprints." This request requires a logged user with the *release-monitorings* role assigned.
